# Agnes API Common Error Codes / Agnes API 常见错误码

Last updated: `2026-06-22 00:00 Asia/Singapore`

This bilingual reference summarizes common Agnes API status codes, likely causes, and recommended fixes. Production-critical behavior, model access, rate limits, and quota rules may change over time. Always confirm current values in the official documentation or platform console.

本文档提供 Agnes API 常见错误码的中英文对照，包括含义、常见原因和建议解决方案。生产环境相关的模型权限、速率限制和配额规则可能随时间变化，请以官方文档或平台控制台为准。

## Quick Reference / 快速对照

| Status | Meaning / 含义 | Common causes / 常见原因 | Recommended fixes / 建议处理 |
| --- | --- | --- | --- |
| `400 Bad Request` | Invalid request parameters. / 请求参数错误，服务器无法处理当前请求。 | Malformed JSON, missing required fields, wrong parameter types, context exceeds model limits, unsupported third-party tool payload shape, invalid image or video parameters. / JSON 格式错误、必填参数缺失、参数类型不正确、上下文超限、第三方工具入参格式不兼容、图片或视频参数异常。 | Validate JSON and required fields, reduce context, start a fresh conversation for long sessions, check model-specific parameters such as size, frame rate, duration, and response format placement. / 检查 JSON 与必填参数，压缩上下文，长对话可新开会话重试，核对尺寸、帧率、时长和 response format 位置等模型参数。 |
| `401 Unauthorized` | Authentication failed. / 鉴权失败，请求未通过身份验证。 | Invalid, expired, missing, or incorrectly configured API key; missing `Authorization` header; API key belongs to the wrong account. / API Key 错误、失效、未正确配置，Header 未携带 Authorization，或使用了错误账号下的 API Key。 | Use `Authorization: Bearer YOUR_API_KEY`, check for copied spaces or newlines, confirm the key is active, and regenerate the key if needed. / 使用正确的 Bearer 格式，检查多余空格或换行，确认 Key 有效，必要时重新生成。 |
| `402 Payment Required` | Balance or quota is insufficient. / 账户余额不足或额度不足。 | Insufficient balance, insufficient Token Plan quota, request cost exceeds available quota, subscription or recharge has not taken effect. / 余额不足、Token Plan 额度不足、请求消耗超过可用额度、订阅或充值未生效。 | Check balance, Token Plan, and subscription status; recharge, upgrade, or reduce request cost by shortening context or lowering image/video specs. / 检查余额、Token Plan 和订阅状态，充值或升级，或通过缩短上下文、降低生成规格减少消耗。 |
| `403 Forbidden` | Account or API key does not have access. / 当前账号或 API Key 没有访问权限。 | Model access is not enabled, enterprise verification or subscription has not taken effect, source network is blocked by policy, or IP/region/network is restricted. / 未开通模型权限、企业认证或订阅未生效、请求来源被策略拦截，或 IP、地区、网络环境受限。 | Confirm account and model permissions, ensure the key belongs to the correct account, retry from another network, and contact support if access should be enabled. / 确认账号与模型权限，检查 Key 是否属于正确账号，更换网络后重试；确认有权限但仍失败时联系支持。 |
| `404 Not Found` | Endpoint, path, model, or resource was not found. / 请求的接口地址、路径、模型或资源不存在。 | Wrong API address, wrong Base URL, wrong endpoint path, duplicated `/v1`, invalid model name, or missing task/resource. / API 地址错误、Base URL 配置错误、路径错误、第三方工具重复拼接 `/v1`、模型名称错误或资源不存在。 | Confirm Base URL, endpoint path, model name, and resource ID. Make sure tools do not append `/v1` twice. / 检查 Base URL、接口路径、模型名和资源 ID，确认第三方工具没有重复拼接 `/v1`。 |
| `405 Method Not Allowed` | The endpoint does not support the HTTP method. / 当前接口不支持使用该请求方法。 | Using `GET` where `POST` is required, using `POST` where `GET` is required, or a third-party tool is configured with the wrong method. / 应使用 POST 却使用 GET，应使用 GET 却使用 POST，或第三方工具请求方法配置错误。 | Check the endpoint documentation and update the client or third-party tool to use the required method. / 查看接口文档并修改客户端或第三方工具的请求方法。 |
| `408 Request Timeout` | Request timed out before completion. / 请求超时，服务器未能在限定时间内完成处理。 | Unstable local network, oversized payload, long context, image/video/long-text tasks, or client timeout is too short. / 本地网络不稳定、请求内容过大、上下文过长、图片/视频/长文本任务耗时较长，或客户端超时时间过短。 | Retry, check network, reduce input size, lower generation specs, increase client timeout, and add retry logic for long tasks. / 重试并检查网络，缩短输入或降低生成规格，适当增加客户端超时时间，并为长任务加入重试机制。 |
| `409 Conflict` | Request conflicts with the current resource or task state. / 当前请求与已有状态冲突。 | Duplicate task submission, same request already in progress, resource state does not allow the operation, or a tool repeatedly submits requests. / 重复提交同一任务、相同请求正在处理中、资源状态不允许当前操作，或第三方工具重复发起请求。 | Avoid duplicate submissions, wait for the current task to finish, check task status, and add client-side duplicate-submit protection. / 避免短时间重复提交，等待当前任务完成，检查任务状态，并在客户端增加防重复提交逻辑。 |
| `413 Payload Too Large` | Request body is too large. / 请求体过大，超过服务器可处理范围。 | Too much context, oversized Base64 image, upload exceeds limits, or too much content sent at once. / 输入上下文过长、Base64 图片过大、上传文件或图片超过限制，或一次性提交内容过多。 | Compress payload, reduce context, split requests, lower upload size or resolution, and prefer public image URLs over large Base64 payloads when possible. / 压缩请求内容，减少上下文，分批提交，降低上传体积或分辨率；可行时优先使用公网图片 URL。 |
| `415 Unsupported Media Type` | File format or `Content-Type` is unsupported. / 请求中的文件格式或 Content-Type 不被支持。 | Unsupported file format, incorrect `Content-Type`, unsupported image/audio/video format, or a third-party tool sets headers incorrectly. / 上传文件格式不支持、Content-Type 设置错误、图片/音频/视频格式不符合要求，或第三方工具自动设置了错误 Header。 | Check supported formats and headers. Use common image formats such as PNG, JPG, JPEG, or WEBP where applicable. / 检查支持格式和 Header；图片建议使用 PNG、JPG、JPEG、WEBP 等常见格式。 |
| `422 Unprocessable Entity` | Request format is valid, but parameter values are not acceptable. / 请求格式正确，但参数内容不符合模型或接口要求。 | Invalid parameter values, image-edit or video parameters outside allowed range, invalid seed, inaccessible image URL, malformed Base64, or unsupported generation size. / 参数内容不合法、图片编辑或视频参数异常、seed/尺寸/比例/帧率超出范围、图片 URL 无法访问、Base64 异常或生成尺寸不符合要求。 | Read the exact error message, validate image/video parameters, check seed range, confirm image URLs are publicly accessible, and validate Base64 content. / 查看具体报错信息，检查图片或视频参数、seed 范围、图片 URL 公网可访问性和 Base64 完整性。 |
| `429 Too Many Requests` | Rate limit exceeded. / 请求频率过高，超过当前账号的 RPM 限制。 | RPM exceeded, too many concurrent requests, repeated requests in a short period, or automatic retries are too aggressive. / RPM 超限、并发请求过多、短时间内重复请求，或自动重试频率过快。 | Wait before retrying, reduce request rate and concurrency, use exponential backoff, avoid tight polling loops, and consider Token Plan for higher throughput. / 等待后重试，降低频率和并发，使用指数退避，避免密集轮询；高请求量场景可考虑 Token Plan。 |
| `431 Request Header Fields Too Large` | Request headers are too large. / 请求 Header 过大，服务器无法处理。 | Oversized custom headers, abnormal authorization value, excessive cookies, or third-party tools attach too many headers. / 自定义 Header 过长、Authorization 异常、Cookie 或自定义 Header 过大，或第三方工具附带过多 Header。 | Remove unnecessary headers and cookies, verify the authorization header, and simplify third-party tool header configuration. / 移除不必要的 Header 和 Cookie，检查 Authorization，并简化第三方工具的请求头配置。 |
| `499 Client Closed Request` | Client closed the connection before completion. / 客户端主动断开连接，请求未完成。 | Client timeout, browser/tool cancellation, local network switch, long task wait, or third-party tool cancels early. / 客户端超时、浏览器或工具中断、本地网络切换或断开、长任务等待过久，或第三方工具提前取消请求。 | Retry, check network stability, increase client timeout, avoid closing pages or tools during execution, and use async polling for long tasks. / 重新请求，检查网络稳定性，增加客户端超时时间，任务执行中不要关闭页面或工具，长任务建议使用异步或轮询机制。 |
| `500 Internal Server Error` | Server-side error or request triggered an internal failure. / 服务器内部错误，或请求参数触发了服务异常。 | Invalid parameters, unsupported image/video dimensions, missing context settings in third-party tools, or temporary upstream issues. / 请求参数异常、图片或视频生成尺寸不符合要求、第三方工具未设置上下文参数，或上游服务临时异常。 | Check parameters, ensure image dimensions meet model requirements, ensure video dimensions meet model requirements, simplify the payload, and retry. / 检查参数，确认图片和视频尺寸符合模型要求，简化请求后重试。 |
| `502 Bad Gateway` | Gateway or proxy could not get a valid upstream response. / 网关或代理服务未能从上游服务获得有效响应。 | Local network issues, DNS problems, proxy configuration issues, unstable route, or temporary upstream fluctuation. / 本地网络异常、DNS 解析异常、代理配置异常、网络链路不稳定，或上游服务临时波动。 | Check network and proxy settings, try another network, update DNS, and retry later. / 检查网络和代理配置，更换网络，修改 DNS，并稍后重试。 |
| `503 Service Unavailable` | Service is temporarily unavailable or a tool routed to the wrong model. / 服务暂时不可用，或第三方工具配置异常导致请求失败。 | Wrong model name, model list not refreshed, no fallback model selected, routing disabled, tool sends request to the wrong model, or short-term service unavailability. / 模型名称错误、未成功获取模型列表、未选择兜底模型、未开启路由、第三方工具请求到错误模型，或服务短时间不可用。 | Check model name, refresh the model list, select a fallback model, enable routing if required, review third-party tool settings, and retry later. / 检查模型名，重新获取模型列表，选择兜底模型，必要时开启路由，检查第三方工具配置并稍后重试。 |
| `504 Gateway Timeout` | Gateway timed out waiting for upstream service. / 网关等待上游服务响应超时。 | Upstream model timeout, image/video/long-context task is slow, unstable route, high service load, or request complexity is too high. / 上游模型响应超时、图片/视频/长上下文任务耗时过长、网络链路不稳定、高峰期负载较高，或请求复杂度过高。 | Retry later, shorten input, lower image/video specs, reduce concurrency, increase client timeout, and add retry logic for long tasks. / 稍后重试，缩短输入，降低生成规格，减少并发，适当增加客户端超时时间，并为长任务加入重试机制。 |
| `520 Unknown Error` | Unknown gateway or upstream error. / 未知错误，通常与网络链路、网关或上游服务波动有关。 | Network route issue, temporary upstream fluctuation, proxy/DNS issue, or gateway returned a non-standard error. / 网络链路异常、上游服务临时波动、代理或 DNS 异常，或网关返回非标准错误。 | Retry with backoff, check network, proxy, and DNS settings, and provide sanitized request metadata to support if it persists. / 使用退避重试，检查网络、代理和 DNS；如持续出现，请向支持提交脱敏后的请求信息。 |
| `522 Connection Timed Out` | Gateway connection to upstream service timed out. / 网关连接上游服务超时。 | Network connection timeout, gateway-to-upstream connection issue, local network problem, proxy issue, or DNS issue. / 网络连接超时、网关到上游服务连接异常、本地网络异常、代理配置异常或 DNS 配置异常。 | Retry, check local network, proxy, and DNS settings, or try another network. / 重试，检查本地网络、代理和 DNS 设置，或更换网络环境。 |
| `524 Timeout Occurred` | Connection succeeded, but upstream processing took too long. / 请求已连接成功，但上游服务处理时间过长导致超时。 | Long-context task, image generation, video generation, high service load, or request complexity is too high. / 长上下文任务、图片生成、视频生成、服务负载较高，或请求复杂度过高。 | Retry later, reduce request complexity, shorten context, lower generation specs, reduce concurrency, and use async polling for long tasks. / 稍后重试，降低请求复杂度，缩短上下文，降低生成规格，减少并发，并对长任务使用异步或轮询机制。 |

## Integration Tips / 接入建议

- Keep API keys in server-side environment variables. Never expose keys in client-side code, screenshots, public issues, or public repositories.
- Store sanitized request examples for debugging: model, endpoint, timestamp, request ID if available, status code, and response body with secrets removed.
- Add exponential backoff for transient errors such as `408`, `429`, `500`, `502`, `503`, `504`, `520`, `522`, and `524`.
- For video generation, create the task first and poll the result with `video_id`:

```text
GET https://apihub.agnes-ai.com/agnesapi?video_id=<VIDEO_ID>
```

- For image and video requests, validate size, aspect ratio, frame rate, duration, public URL accessibility, and Base64 payload size before retrying.
